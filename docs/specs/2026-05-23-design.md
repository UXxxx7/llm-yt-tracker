# LLM YouTube Landscape Tracker — Design Spec

Date: 2026-05-23
Status: Approved (sections 1–4)

## 1. Goal

Monitor a curated list of LLM-focused YouTube channels, transcribe new videos with AI, extract what each creator **actually says about LLM topics** (not titles or thumbnails), and publish a categorised, continuously-updating table on a public URL.

This is a recruitment exercise. The reviewer will:
- (A) open the live URL in a browser
- (B) listen to a walkthrough of data-collection → AI summary → auto-update
- (C) verify the table updates as new videos appear

## 2. Constraints

- Public URL, no login required.
- Auto-updating: new videos appear within ~24h with zero manual intervention.
- Summaries must be derived from transcripts, not titles/thumbnails/descriptions. Each row carries `transcript_source` and `transcript_excerpt_chars` as proof.
- Monthly cost target: <$5.
- Runtime host: this Windows machine (always-on). Schedule via Windows Task Scheduler.

## 3. Stack

- **OpenClaw skill** (thin wrapper) at `C:\Users\11989\.openclaw\workspace\skills\llm-yt-tracker\SKILL.md`. The skill body documents purpose and shells out to `python -m pipeline.run`.
- **Pipeline (Python 3.13)** in `D:\YMX\llm-yt-tracker\pipeline\`:
  - YouTube Data API v3 for new-video discovery
  - yt-dlp for captions (and audio download when needed)
  - OpenAI Whisper API as transcription fallback
  - DeepSeek-V4 (via OpenAI-compatible SDK at `https://api.deepseek.com`) for summary + tagging
- **Data**: `web/videos.json` (table data, git-tracked), `pipeline/state.json` (dedupe / incremental state, git-tracked).
- **Frontend**: single `web/index.html` + `web/app.js` + Tailwind CDN. Reads `videos.json`, renders sortable/filterable table.
- **Deployment**: GitHub Pages from the `main` branch's `web/` folder. Cron commits + pushes; Pages auto-redeploys.
- **Schedule**: Windows Task Scheduler triggers `python -m pipeline.run` every 6 hours and at startup.

### Hard field constraints (reviewer will spot-check)

- `transcript_source` ∈ {`"captions"`, `"whisper"`}. No `"description"` / `"title"` allowed.
- `transcript_excerpt_chars` ≥ 1000, else the video is dropped (not written).
- `summary` must be produced by the LLM from `transcript_excerpt`. The prompt explicitly forbids referencing title/description.
- `topic_tags` must come from the controlled vocabulary below; 1–5 tags; `["uncategorized"]` if none fit.

## 6. Module contracts

### pipeline/fetch_videos.py
- Input: `channels.yaml`, `state.json`, `YOUTUBE_API_KEY`
- Output: `list[dict]` with `video_id / channel / channel_id / title / published_at / url / duration_seconds`
- Drops live streams and Shorts (duration < 90s) and already-processed IDs.
- Per-channel failures are logged and skipped — they do not abort the run.
- Interface: `fetch_new_videos(channels: list, state: dict) -> list[dict]`

### pipeline/transcribe.py
- Input: `video_id`, `url`
- Output: `{"source": "captions"|"whisper", "text": str, "chars": int}` or `None`
- Strategy:
  1. `yt-dlp --write-auto-subs --skip-download --sub-lang en` → if captions ≥ 1000 chars, return `captions`
  2. Else `yt-dlp -x --audio-format mp3` → OpenAI Whisper API → return `whisper`
  3. Temp audio file deleted on exit, regardless of outcome
- Interface: `transcribe(video_id: str, url: str) -> dict | None`

### pipeline/summarise.py
- Input: transcript text, controlled-tag list, known-channel list
- Output: `{"summary": str, "topic_tags": list[str], "key_quotes": list[str], "related_channels": list[str]}`
- DeepSeek-V4 call via OpenAI SDK (`base_url=https://api.deepseek.com`); response forced to JSON.
- Prompt constraints (must appear verbatim in the prompt body):
  - "Base your summary ONLY on the transcript below. Do not use the title or any outside knowledge."
  - "Choose topic_tags only from this list: <controlled vocabulary>."
  - "summary must be 2–3 sentences in Chinese summarising what the speaker actually said."
- 2 retries on API errors (exponential backoff). 1 retry with stricter prompt on JSON parse failure.
- Interface: `summarise(transcript: str, controlled_tags: list[str], known_channels: list[str]) -> dict`

### pipeline/build_table.py
- Input: enriched new entries, existing `web/videos.json`, existing `state.json`
- Output: new `web/videos.json`, new `state.json`
- Dedup by `video_id` (new overrides old); sort by `published_at` desc; update state.
- Interface: `build_and_save(new_entries: list[dict])`

### pipeline/publish.py
- Behaviour: `git add web/videos.json pipeline/state.json` → if diff exists, `git commit -m "data: refresh <ISO timestamp>"` → `git push`.
- Network/conflict failures are logged; local changes preserved; next cron retries.
- Interface: `publish() -> bool`

### pipeline/run.py
Orchestrates the steps above. Each video is processed in a `try/except` so one failure cannot abort the rest. The whole `main()` is also wrapped to guarantee exit code 0 (failures go to `logs/cron.log`).

### SKILL.md
YAML frontmatter (`name`, `description`) + body. Body documents purpose, trigger phrases (`刷新 LLM 表格`, `refresh llm tracker`), the exact command (`cd D:\YMX\llm-yt-tracker && python -m pipeline.run`), where to find logs, and a failure checklist.

### web/app.js
Loads `videos.json`, renders table with: multi-select tag filter, channel dropdown, keyword search, sortable headers, expandable rows showing `summary` + `key_quotes` + first 500 chars of `transcript_excerpt`. Zero JS deps (Tailwind via CDN).

### scripts/register_task.ps1
PowerShell script that registers a Windows Scheduled Task: trigger every 6h + at startup; action runs `python -m pipeline.run` in `D:\YMX\llm-yt-tracker`; stdout/stderr → `logs\cron.log`.

## 7. Error handling

| Failure | Behaviour | User-visible outcome |
|---|---|---|
| Single-channel YouTube API failure | Log warning, skip channel | That channel un-updated this cycle |
| YouTube quota exhausted | Log error, run exits 0 | No new data this cycle |
| Caption fetch fails for a video | Auto-fallback to Whisper | Transparent |
| Whisper also fails | Log warning, video dropped | Video not in table, retried next cycle |
| Transcript <1000 chars | Drop video (likely Shorts/music) | Video not in table |
| DeepSeek API error | 2 retries with backoff, then drop video | Video not in table this cycle |
| DeepSeek returns invalid JSON | 1 retry with stricter prompt, then drop | Same as above |
| `git push` fails | Log error, preserve local diff | Live URL stale until next cycle |
| `state.json` corrupted | Rebuild from `videos.json` | Self-heals once |

Core rule: cron always exits 0. Every failure is swallowed, logged to `logs/cron.log`, and retried next cycle.

## 8. Testing

Pragmatic, not exhaustive:

1. `tests/test_summarise.py` — mock DeepSeek with valid JSON, JSON-in-markdown-fence, missing fields, illegal tags; verify each case.
2. `tests/test_build_table.py` — construct old `videos.json` + new entries; assert dedup + sort + state correctness.
3. `tests/test_transcribe.py` — mock yt-dlp subprocess; verify the three branches (captions / whisper / both fail).
4. `tests/fixtures/` — 2–3 real transcript samples + hand-labelled expected summaries (smoke).
5. End-to-end smoke: `python -m pipeline.run --dry-run --limit 1` (no disk write, no push) to verify API keys.

## 9. Walkthrough script (review)

10 min, mapped to Assessment (A)/(B)/(C):

**(A) Live URL — 2 min.** Open `https://<user>.github.io/llm-yt-tracker/`. Demo tag filter (click "RAG"), channel filter, keyword search, header sort. Expand a row → summary + quotes + transcript excerpt.

**(B) Three pipelines — 5 min.**
1. **Collection**: open `pipeline/fetch_videos.py` — YouTube API + incremental dedup. Open `pipeline/transcribe.py` — captions-first / Whisper-fallback, cost rationale.
2. **AI summary**: open `pipeline/summarise.py` — point at the prompt line forbidding title/description use. Open `videos.json`, pick a row, show `transcript_excerpt` next to `summary` to prove the summary came from the transcript.
3. **Auto-update**: Windows Task Scheduler view → schedule. Tail `logs/cron.log`. Open GitHub commit history → one `data: refresh ...` commit every ~6h. Pages auto-redeploys.

**(C) Stays current — 3 min.** Show latest auto-commit timestamp. Trigger the OpenClaw skill live (`刷新 LLM 表格`) or run `python -m pipeline.run` directly. Wait 1–2 min, refresh the page, show new rows (or the "no new videos" log explaining incremental logic). Show `state.json` and explain why videos aren't re-transcribed.

## 11. Out of scope

- Comment monitoring or sentiment analysis
- Multi-language transcripts beyond English source + Chinese summary
- A database (JSON is sufficient at this scale)
- Authenticated access
- A second "interactive analysis" OpenClaw skill (could be added later)
